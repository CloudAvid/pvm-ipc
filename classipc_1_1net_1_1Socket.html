Detailed Description

Responsible for creating and handling sockets.

In all used syscalls in this class; as error occurrence, logs will be printed in debug or warning mode includes errno message; unless an exception is thrown.

Note
errno will be set for all methods wchich using syscalls.
CLOEXEC flag is set for fd by defualt, in rare cases which is needed to be removed use and unsetFDFlags(); as shown below:
// after socket creation.
unsetFDFlags(FD_CLOEXEC);
Warning
unseting CLOEXEC is not recommanded.
Note
by default we assume socket is block. However methods ,who use timeout, set nonblock flag for fd explicity then unset it as return.
Warning
after creating a socket (successful return from bind() or connect()), you MUST close it explicity.
Note
Datagram transmissions (SOCK_DGRAM) in unix domain (AF_UNIX) is reliable, in spite of IPv4 (AF_INET) and IPv6 (AF_INET6) domains.
